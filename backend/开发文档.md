# 政策雷达-电力 后台管理系统开发文档

## 项目概述

- **项目名称**: 政策雷达-电力
- **技术栈**: Go + Gin + GORM (后端), Vue 3 + TypeScript + Element Plus + Tailwind CSS (前端)
- **数据库**: SQLite
- **主题色**: #DB373B
- **设计风格**: 简洁风格，2px 圆角边框

## 功能模块

| 模块 | 说明 |
|------|------|
| 国标政策管理 | 国家标准政策的 CRUD、核验、批量导入 |
| 行标政策管理 | 行业标准政策的 CRUD、核验、批量导入 |
| 地标政策管理 | 地方标准政策的 CRUD、核验、批量导入 |
| 政策分类管理 | 政策分类的 CRUD，关联政策数量统计 |
| 推荐政策管理 | 前台推荐政策的配置，Markdown 编辑器 |
| 爬虫热更新 | Python 爬虫脚本编辑(Monaco Editor)、执行、数据导入 |
| 数据仪表盘 | 统计卡片、饼图、柱状图、最近政策列表 |

## 爬虫脚本设计

### 国标爬虫 (nationalScript)

**数据源**:
- 接口1: `std.samr.gov.cn/gb/search/gbQueryPage` (JSON API)
- 接口2: `openstd.samr.gov.cn/bzgk/gb/std_list` (HTML解析)
- 详情页: `openstd.samr.gov.cn/bzgk/gb/newGbInfo?hcno=xxx`

**接口1参数**:
```python
params = {
    'searchText': keyword,
    'ics': '',
    'state': '',
    'ISSUE_DATE': '-36',  # 近36个月
    'sortOrder': 'asc',
    'pageSize': 50,
    'pageNumber': page,
}
```

**接口1字段映射**:
| API字段 | 数据库字段 |
|---------|-----------|
| C_STD_CODE | standard_no |
| C_C_NAME / C_STD_NAME | standard_name |
| ISSUE_DATE | publish_date |
| ACT_DATE | implement_date |
| STATE | status |
| STD_NATURE | nature |

**接口2解析**:
- HTML表格提取: 标准号、名称、状态、发布日期、实施日期、hcno
- 详情页获取: `english_name`, `ccs_code`, `ics_code`, `department`, `technical_dept`, `publisher`

**合并逻辑**:
```python
# 按标准号取并集
all_std_nos = set(api1_map.keys()) | set(api2_map.keys())
# 优先使用接口2数据，补充详情页字段
```

**特点**:
- stderr 输出日志，stdout 输出 JSON 数据
- Upsert 导入（standard_no + standard_name 唯一）
- 详情页延迟 0.2s 避免请求过快

### 行标爬虫 (industryScript)

**数据源**: `hbba.sacinfo.org.cn/stdQueryList`

**参数**:
```python
data = {
    'current': page,
    'size': 100,
    'key': keyword,
    'industry': '电力',
    'pubdate': '-36',  # 近36个月
    'status[]': ['即将实施', '现行']
}
```

**特点**:
- POST 请求，form-data 格式
- 时间戳转换 (毫秒 → 日期)
- 支持 `replace_standard` 替代标准字段
