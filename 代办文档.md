## 二、完整产品需求

### （1）后台管理端

#### 1. 页面模块

- 国标政策管理页面
- 行标政策管理页面
- 地标政策管理页面（表结构同国标/行标）
- 政策分类管理页面
- 政策热更新页面

#### 2. 政策管理页面（国标/行标/地标 通用功能）

1. 基础功能：**增删改查、分页、条件搜索**
2. 搜索字段：标准号、标准名称（支持模糊搜索）
3. 表单字段：支持录入/编辑**政策下载链接**、基础政策信息
4. 核心状态：**人工核验状态流转**（待核验/已核验/不通过）
5. 批量能力：支持Excel批量导入政策数据
6. 列表展示：展示所有字段、核验状态、下载链接、创建/更新时间

#### 3. 政策分类管理页面

1. 展示分类名称、分类编码等基础信息
2. 展示**当前分类关联的政策总数量**
3. 支持分类的增删改查、关联政策查看

#### 4. 政策热更新页面（数据爬取+导入）

1. 集成**代码编辑器插件**（支持脚本编写、格式化、高亮）
2. 功能拆分：独立爬取脚本模块 → 输入查询内容 → 执行脚本 → 返回结构化数据 → 预览 → 一键导入数据库
3. 数据校验：自动匹配政策表结构，结构合规方可导入
4. 日志记录：展示脚本执行结果、导入成功/失败条数

1. **首页**：展示政策轮播图、推荐政策解析入口
2. **政策列表页**：支持标准号/标准名称搜索、分页展示所有政策
3. **政策详情页**：展示政策完整信息、下载链接

### （3）数据库表结构设计（强制规范）

#### 1. 政策主表（3张，结构完全一致）

- `national_standard`（国标政策表）
- `industry_standard`（行标政策表）
- `local_standard`（地标政策表）

**表字段（基于datas目录Excel扩展）**：

- 主键id、Excel原有所有字段、`standard_type`（标准类型：国标/行标/地标）
- `download_url`（政策下载链接）、`check_status`（核验状态：0待核验/1已核验/2不通过）
- `create_time`（创建时间）、`update_time`（更新时间）、`category_id`（关联分类ID）

#### 2. 辅助表

1. `policy_category`（政策分类表）：id、分类名称、分类编码、创建时间、更新时间
2. `policy_recommend`（推荐政策解析表）：id、政策ID、解析标题、解析内容、排序、创建时间、更新时间（用于首页轮播详情）
