> **版本**：v1.0-beta（小版本）

> **文档类型**：项目功能与数据模型说明

## 一、项目概述

**Policy Radar（政策雷达）** 是一个电力行业标准政策管理与展示系统，提供政策数据的爬取、管理和展示功能。系统核心定位为政策溯源与展示，旨在帮助用户及时追踪最新政策动态，清晰了解政策更新内容及其对自身业务的影响范围。

### 当前版本实现范围

本版本（v1.0-beta）已实现：

- ✅ 国家标准、行业标准、地方标准的数据管理
- ✅ Python 爬虫数据采集（国标、行标、地标）
- ✅ PC 端管理后台（政策录入、导入）
- ✅ H5 移动端展示页面
- ✅ 爬虫热更新功能（在线执行 Python 脚本）
- ✅ 政策分类与推荐管理

### 项目定位

- **管理端**：PC Web 后台管理系统，用于政策数据的录入、管理、分类和核验
- **展示端**：H5 移动端页面，用于政策数据的浏览、搜索和推荐展示
- **数据源**：通过 Python 爬虫自动采集国家标准、行业标准等数据

## 四、数据爬取模块详解

### 4.1 爬虫脚本说明

项目包含 3 个主要爬虫脚本，位于 `Datas/` 目录：

| 脚本文件                | 数据源                | 采集目标                  |
| ----------------------- | --------------------- | ------------------------- |
| `crawler.py`            | `std.samr.gov.cn`     | 国家标准（API 接口方式）  |
| `crawler_guobiao_v2.py` | `openstd.samr.gov.cn` | 国家标准（HTML 页面方式） |
| `crawler_hangye.py`     | `hbba.sacinfo.org.cn` | 行业标准                  |

### 4.2 爬虫依赖环境

**Python 依赖包**：

```bash
pip install requests pandas beautifulsoup4 openpyxl
```

| 依赖包                | 用途                         |
| --------------------- | ---------------------------- |
| `requests`            | 发送 HTTP 请求获取网页数据   |
| `BeautifulSoup` (bs4) | 解析 HTML 页面结构，提取数据 |
| `pandas`              | 数据处理和 DataFrame 操作    |
| `openpyxl`            | 导出数据到 Excel 文件        |

### 4.3 爬虫工作流程

#### 国家标准爬虫（crawler.py）

1. 调用 API：`https://std.samr.gov.cn/gb/search/gbQueryPage`
2. 参数：`searchText=电力`，按页码分页获取
3. 解析 JSON 返回数据，提取标准基本信息
4. 访问详情页获取更多信息（归口单位、起草单位、采标情况）
5. 导出 Excel：`电力标准数据.xlsx`

#### 国家标准爬虫 V2（crawler_guobiao_v2.py）

1. 访问页面：`https://openstd.samr.gov.cn/bzgk/gb/std_list`
2. POST 参数：`p.p2=电力` 搜索关键词
3. 解析 HTML 表格结构提取数据
4. 访问详情页补充字段信息
5. 导出 Excel：`国家标准电力数据_接口2.xlsx`

#### 行业标准爬虫（crawler_hangye.py）

1. POST 请求：`https://hbba.sacinfo.org.cn/stdQueryList`
2. 参数：`key=电力`, `industry=电力`
3. 解析 JSON 返回的 `records` 数组
4. 访问详情页获取完整信息
5. 导出 Excel：`行业标准电力数据.xlsx`

### 4.4 爬虫热更新机制

后端提供 `/api/v1/crawler/execute` 接口，支持在线执行 Python 脚本：

1. 前端通过 Monaco Editor 编辑 Python 爬虫脚本
2. 后端接收脚本内容，使用 `exec.Command("python3", "-c", script)` 执行
3. 解析脚本输出的 JSON 数据
4. 记录执行日志到 `crawler_log` 表
5. 返回执行结果给前端展示

**安全机制**：后端会检查脚本是否包含危险代码（如 `os.system`, `subprocess.call`, `eval`, `exec`）

| 表名                | 中文名     | 说明                 | 关联关系             |
| ------------------- | ---------- | -------------------- | -------------------- |
| `national_standard` | 国家标准表 | 存储国家标准政策数据 | 关联 policy_category |
| `industry_standard` | 行业标准表 | 存储行业标准政策数据 | 关联 policy_category |
| `local_standard`    | 地方标准表 | 存储地方标准政策数据 | 关联 policy_category |

**索引说明**：

- `code` 字段建有**唯一索引**，确保分类代码唯一
- `deleted_at` 字段建有索引

### 5.6 推荐政策表（policy_recommend）

**表名**：`policy_recommend`

**用途**：配置首页展示的推荐政策，支持按排序展示。

| 字段名       | Go 类型        | 数据库类型 | 长度/约束       | JSON键        | 说明                         | 示例值                       |

### 5.9 字段设计说明

#### 时间字段处理

- `CreatedAt`、`UpdatedAt`：GORM 自动管理，不输出到 JSON
- JSON 输出时通过自定义 `MarshalJSON` 方法格式化为 `"2024-03-31 15:30:00"` 格式
- 使用 `time.Time` 类型存储，便于时区处理和排序

#### 软删除机制

- 所有核心表（除 crawler_log）都使用 GORM 的软删除功能
- `DeletedAt` 字段记录删除时间，不输出到 JSON
- 查询时自动过滤已删除记录，需使用 `Unscoped()` 才能查询到

#### 核验状态设计

- `CheckStatus` 字段用于人工核验爬取数据的准确性
- 0 = 待核验（默认值，新导入数据）
- 1 = 已核验（人工确认数据正确）
- 2 = 不通过（数据有问题，需修正）

#### 数据来源标记

- `Source` 字段区分数据来源
- `crawl` = 爬虫自动采集（需核验）
- `manual` = 人工录入（默认已核验）
