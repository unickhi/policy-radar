# 政策雷达-电力系统 开发设计文档

## 一、项目概述

政策雷达-电力系统是一个电力行业政策标准管理平台，包含后台管理和前端展示两大模块。

### 技术栈

| 组件 | 技术 | 版本 |
|------|------|------|
| 前端框架 | Vue 3 + TypeScript | ^3.4 |
| UI组件库 | Element Plus | ^2.4 |
| 构建工具 | Vite | ^5.0 |
| 状态管理 | Pinia | ^2.1 |
| 路由 | Vue Router | ^4.2 |
| 后端框架 | Go + Gin | ^1.21 |
| ORM | GORM | ^1.25 |
| 数据库 | MySQL / SQLite | 8.0 / 3 |

### 主题色规范

| 用途 | 颜色值 |
|------|--------|
| 主色 | `#DB373B` |
| 辅助色 | `#FEF2F2` |
| 底色 | `#FFFFFF` |
| 按钮圆角 | `5px` |

## 四、页面交互设计

### 4.1 政策管理页面（国标/行标/地标通用）

#### 搜索区域
- 标准号输入框（精确搜索）
- 标准名称输入框（模糊搜索）
- 核验状态下拉框（待核验/已核验/不通过）
- 标准状态下拉框（现行/即将实施/废止）
- 搜索按钮、重置按钮

#### 操作区域
- 新增按钮（打开新增弹窗）
- 批量导入按钮（上传Excel）
- 批量删除按钮（选中后可用）

#### 数据表格
- 列字段：标准号、标准名称、发布日期、实施日期、状态、核验状态、下载链接、操作
- 核验状态：Tag标签显示（待核验-灰色、已核验-绿色、不通过-红色）
- 操作列：编辑、删除、核验、下载

#### 弹窗表单
- 新增/编辑弹窗：所有字段表单
- 核验弹窗：选择核验状态、填写备注
- 导入弹窗：文件上传、导入预览

### 4.2 政策分类管理页面

#### 数据表格
- 列字段：分类名称、分类编码、关联政策数量、操作
- 关联政策数量：分国标/行标/地标统计显示

#### 操作区域
- 新增分类、编辑分类、删除分类

### 4.3 政策热更新页面

#### 代码编辑器区域
- Monaco Editor / CodeMirror 编辑器
- 支持Python语法高亮
- 预置爬取脚本模板

#### 执行控制区域
- 查询内容输入
- 执行按钮、停止按钮
- 数据源选择（国标接口1/国标接口2/行标接口）

#### 数据预览区域
- 表格展示爬取结果
- 字段映射配置
- 数据校验提示

#### 导入操作区域
- 一键导入按钮
- 导入日志展示（成功/失败条数）

## 五、API接口设计

### 5.1 政策管理接口

```
GET    /api/v1/national          # 国标列表
GET    /api/v1/national/:id      # 国标详情
POST   /api/v1/national          # 新增国标
PUT    /api/v1/national/:id      # 更新国标
DELETE /api/v1/national/:id      # 删除国标
POST   /api/v1/national/import   # 批量导入
PUT    /api/v1/national/:id/check # 核验状态更新

GET    /api/v1/industry          # 行标列表
GET    /api/v1/industry/:id      # 行标详情
POST   /api/v1/industry          # 新增行标
PUT    /api/v1/industry/:id      # 更新行标
DELETE /api/v1/industry/:id      # 删除行标
POST   /api/v1/industry/import   # 批量导入
PUT    /api/v1/industry/:id/check # 核验状态更新

GET    /api/v1/local             # 地标列表
GET    /api/v1/local/:id         # 地标详情
POST   /api/v1/local             # 新增地标
PUT    /api/v1/local/:id         # 更新地标
DELETE /api/v1/local/:id         # 删除地标
POST   /api/v1/local/import      # 批量导入
PUT    /api/v1/local/:id/check   # 核验状态更新
```

### 5.2 分类管理接口

```
GET    /api/v1/categories        # 分类列表
GET    /api/v1/categories/:id    # 分类详情
POST   /api/v1/categories        # 新增分类
PUT    /api/v1/categories/:id    # 更新分类
DELETE /api/v1/categories/:id    # 删除分类
GET    /api/v1/categories/:id/policies # 分类关联政策
```

### 5.3 热更新接口

```
POST   /api/v1/crawler/execute   # 执行爬取脚本
GET    /api/v1/crawler/logs      # 获取执行日志
POST   /api/v1/crawler/import    # 导入爬取数据
```

### 5.4 前端展示接口

```
GET    /api/v1/policies          # 全部政策列表（合并三表）
GET    /api/v1/policies/:id      # 政策详情
GET    /api/v1/recommends        # 推荐政策列表
GET    /api/v1/search            # 政策搜索
```

## 七、开发优先级

1. **P0 - 基础架构**：项目框架搭建、数据库表创建、基础API
2. **P1 - 后台管理**：国标/行标/地标政策管理页面（CRUD）
3. **P2 - 分类管理**：分类管理页面、关联统计
4. **P3 - 批量导入**：Excel批量导入功能
5. **P4 - 热更新**：代码编辑器、爬取脚本集成
6. **P5 - 前端展示**：首页、政策列表、政策详情
